Let's continue working on our first automation.
If you are not already there, go to Power Automate.

Navigate to Solutions in the left nav, click on your New Hire Onboarding solution.

Edit the flow: When a new employee onboarding input form is submitted.

In this section, we will be working on populating tasks for the new hire.

  1. After the step: Add user to licensing group
  2. Add a new step
  3. Search for and select Get items (SharePoint)
    1. Site: /sites/HR
    2. List: WFEmployeeOnboardingTasks
  4. Rename to: Get items from WFEmployeeOnboardingTasks
  5. Add a new step
  6. Search for and select: Create item (SharePoint)
    1. Rename this to: Create new employee tasks
    2. Site: /sites/employees
    3. List: NewHireTasks
    4. Title: {Title} from dynamic content
    5. This will automatically create an Apply all around this step
    6. Rename the Apply to: Apply to each WFEmployeeOnboardingTasks
    7. Expand the Create new employee tasks step
    8. AssignedTo Claims:
      1. Custom value
      2. In Dynamic content, scroll down to the Create user in AD section
      3. Select {User Principal Name}
    9. Manager Claims:
      1. Custom value
      2. In Dynamic content, scroll to the When an item is created in onboarding form
      3. Select {HiringManager Claims}
    10. Description:
      1. From WFEmployeeOnboardingTasks section
      2. Select {Description}
    11. Status value: Pending, which is default
    12. DaysToComplete: {DaysToComplete} from Dynamic content WFEmployeeOnboardingTasks section
    13. StartDate:
      1. Search for StartDate in Dynamic content
      2. Select {StartDate} under When an item is created in onboarding form
