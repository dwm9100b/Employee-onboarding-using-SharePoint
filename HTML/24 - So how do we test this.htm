Testing is key to any application and workflows. This is usually the step that most people rarely do as its very time consuming. A good dev will always test every possible permutation of data. So let's give it a shot.

We will create some test cases with expected outcomes based only on the data that we've inputted into the supporting lists. You should also create an external test email at one of the main properties like gmail/outlook.com.

Once you validate that each test case produced the correct results, you should go into each area and delete the populated data. Especially in OnboardingInputForm, OnboardingTracker and NewHireTasks. It would also be a good idea to delete the sent emails so that confusion won't arise on whether they are new or old.

Add a new entry to the OnboardingInputForm

FirstName New1
LastName User1
StartDate Add a date thats 2 weeks from now
WorkLocation HQ
OnboardType W2
HiringManager hiring.mgr

    Key & Fob New1.User1 You
    validate    		 0 You
    validate    		 SG
    Facilities    		 Pending You validate
    Book
    welcome    		 New1.User1 You
    validate    		 0 You

    Calendar
